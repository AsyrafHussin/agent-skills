# Sections

This file defines the section ordering, severity, and summary used by this review lens.

---

## 1. Type Coverage (type-type-coverage)

**Impact:** HIGH
**Description:** Typed public APIs, critical internals, and avoiding implicit any gaps.

## 2. Type Correctness (type-type-correctness)

**Impact:** CRITICAL
**Description:** Annotations, assertions, and guards that match real behavior.

## 3. Type Safety (type-type-safety)

**Impact:** CRITICAL
**Description:** Unsafe casts, unknown data, discriminated unions, and runtime checks.

## 4. Generic Discipline (type-generic-discipline)

**Impact:** MEDIUM
**Description:** Constraints, parameter bounds, and avoiding generic overuse.
